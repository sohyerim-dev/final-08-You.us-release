# 요구사항 정의서

## 1. 문서 정보

### 목적

선물 선택에 어려움을 겪는 사용자를 위해, 간단한 질문(테스트)을 통해 상황과 관계에 맞는 선물을 추천하는 쇼핑몰 서비스를 정의한다.
 본 문서는 서비스의 방향성, 사용자 역할, 요구사항을 명확히 하여 기획·디자인·개발 전반의 기준으로 활용한다.

### 대상

- 서비스 기획자
- 디자이너
- 프론트엔드 개발자
- 프로젝트 이해관계자(멘토, 팀원)

### 범위

- 선물 추천 쇼핑몰의 MVP 기준 정의
- 사용자 흐름 및 핵심 기능 정리
- 실제 결제·물류·CS 시스템의 상세 구현은 범위 외로 둔다

------

## 2. 시스템 개요

### 해결하려는 문제

- 선물을 고르는 과정에서 발생하는 선택 피로
- 상대에 대한 정보 부족으로 인한 선물 실패 불안
- 검색 기반 쇼핑몰의 과도한 선택지와 비교 부담

### 핵심 가치

- 고민을 대신해주는 추천
- 질문에 답하는 것만으로 선물 완성
- 관계·상황 중심의 큐레이션 경험

------

## 5. 제약 및 가정

### 제약

- 외부 결제 시스템은 연동하지 않는다(MVP 기준)
- 실제 물류 시스템과의 실시간 연동은 하지 않는다
- 추천 정확도는 테스트 데이터 기반으로 제한된다

### 가정

- 사용자는 긴 설명보다 질문 기반 인터랙션을 선호한다
- 추천의 “정확성”보다 “고민을 덜어주는 경험”이 중요하다
